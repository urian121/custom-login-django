9.  Vamos a nuestro modelo de nuestra aplicaciÃ³n custom_users, allÃ­ definimos nuestra clase que nos permitirÃ¡ personalizar el modelo usuario de Django.

        from django.contrib.auth.models import AbstractUser
        from django.db import models

        class CustomUser(AbstractUser):
                first_name = models.CharField(max_length=20, blank=True)
                last_name = models.CharField(max_length=20, blank=True)
                twitter_handle = models.CharField(max_length=20, blank=True)

###### AquÃ­ estamos creando un modelo (una tabla en nuestra BD) que hereda todos los campos de la AbstractUser clase que tiene los campos; username, username, email, password, etc, pero tambiÃ©n estamos agregando un par de campos que no estÃ¡n allÃ­. Agregamos los campos: first_name, last_name, twitter_handle
